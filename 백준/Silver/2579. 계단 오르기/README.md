### 문제 설명

<p>계단 오르기 게임은 계단 아래 시작점부터 계단 꼭대기에 위치한 도착점까지 가는 게임이다. <그림 1>과 같이 각각의 계단에는 일정한 점수가 쓰여 있는데 계단을 밟으면 그 계단에 쓰여 있는 점수를 얻게 된다.</p>

	<li>연속된 세 개의 계단을 모두 밟아서는 안 된다. 단, 시작점은 계단에 포함되지 않는다.</li>

<p>각 계단에 쓰여 있는 점수가 주어질 때 이 게임에서 얻을 수 있는 총 점수의 최댓값을 구하는 프로그램을 작성하시오.</p>

### 입력 

 <p>입력의 첫째 줄에 계단의 개수가 주어진다.</p>

<p>둘째 줄부터 한 줄에 하나씩 제일 아래에 놓인 계단부터 순서대로 각 계단에 쓰여 있는 점수가 주어진다. 계단의 개수는 300이하의 자연수이고, 계단에 쓰여 있는 점수는 10,000이하의 자연수이다.</p>

### 출력 

 <p>첫째 줄에 계단 오르기 게임에서 얻을 수 있는 총 점수의 최댓값을 출력한다.</p>

### 🔧 fail log

#### 아이디어
- DP를 사용한다.
- `d[i]`를 `i`번째 계단을 반드시 밟았을 때 얻을 수 있는 최대 점수로 정의한다.
- 마지막 계단 `i`에 도착하는 경우는 두 가지로 나뉜다.
  - `i-2 -> i`로 오는 경우
  - `i-3 -> i-1 -> i`로 오는 경우
- 따라서 점화식은 아래와 같다.
  - `d[i] = max(d[i-2] + steps[i], d[i-3] + steps[i-1] + steps[i])`

#### 알고리즘 판단 근거
- 이 문제는 각 계단까지의 최적값을 바탕으로 다음 상태의 최적값을 구하는 전형적인 DP 문제이다.
- 문제 조건상 마지막 계단은 반드시 밟아야 하고, 세 계단을 연속해서 밟을 수 없다.
- 따라서 `i`번째 계단에 도착하는 합법적인 경우는 아래 두 가지뿐이다.
  - `i-2`에서 두 칸 올라 `i`로 오는 경우
  - `i-3`에서 `i-1`, `i`를 연속해서 밟는 경우
- 이 두 경우 중 최대값을 선택하면 된다.

#### 복잡도
- 시간: `O(n)`
- 공간: `O(n)`

### 오답

#### 오답 유형
- WA 발생
  - WA : wrong answer
  - 문제의 조건을 충분히 고려하지 않았음

#### 오답 원인
- 처음에는 점화식을 아래와 같이 잘못 세웠다.
  - `d[i] = max(d[i-3] + steps[i-2] + steps[i], d[i-2] + steps[i])`
- 겉보기에는 `i-3, i-2, i`가 연속 3계단이 아니므로 가능해 보였다.
- 하지만 `d[i-3]` 내부 경로가 이미 `... -> i-4 -> i-3` 형태일 수 있기 때문에, 여기에 `steps[i-2]`를 더하면 실제 경로가 `... -> i-4 -> i-3 -> i-2 -> i`가 될 수 있다.
- 이 경우 `i-4, i-3, i-2`가 연속 3계단이 되어 조건을 위반한다.
- 즉, `d[i-3] + steps[i-2] + steps[i]`는 항상 안전한 경우만 보장하지 못한다.

#### 해결 방법
- 마지막 계단 `i`를 반드시 밟는 경우를 다시 분리해서 생각했다.
- 가능한 경우는 아래 두 가지뿐이다.
  - `d[i-2] + steps[i]`
  - `d[i-3] + steps[i-1] + steps[i]`
- 두 번째 경우는 `i-2`를 비우는 구조이므로, 어떤 경우에도 세 계단 연속 방문이 발생하지 않는다.
- 점화식을 수정하여 해결할 수 있었다!
